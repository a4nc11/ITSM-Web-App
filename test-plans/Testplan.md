## 1. Introduction
This document defines the testing strategy for the IT Service Management (ITSM) Web Application.
The goal is to ensure reliability, usability, and correctness of core functionalities such as
ticket submission and UI behavior.

---

## 2. Scope of Testing

### In Scope
- UI rendering
- Ticket submission form
- Input validation
- Basic navigation
- Cross-browser compatibility

### Out of Scope
- Backend integrations
- Security penetration testing
- Performance testing

---

## 3. Test Environment
- OS: Windows 10 / 11
- Browser: Google Chrome, Microsoft Edge
- Framework: React (Create React App)
- Tools: Manual Testing, Browser DevTools

---

## 4. Test Types
- Manual Functional Testing
- UI Testing
- Smoke Testing
- Regression Testing

---

## 5. Entry & Exit Criteria

### Entry Criteria
- Application loads successfully
- No build errors

### Exit Criteria
- All critical test cases passed
- No high-severity defects open

---

## 6. Risks
- UI inconsistency across browsers
- User input not properly validated

---

## 7. Approval
This test plan is approved for execution once development is complete.
